# Content Type Rules

Use these rules when processing academic paper content.

## Title, Abstract, Keywords

- Translate fully.
- Keep acronyms and proper nouns where appropriate.
- For keywords, translate each keyword and keep separators readable.

## Body Text

- Translate every paragraph.
- Preserve paragraph order.
- Do not omit caveats, limitations, examples, or parenthetical qualifications.

## Headings

- Translate headings.
- Preserve heading hierarchy.
- Keep section numbers when present.

## Page Markers

- Preserve page markers exactly, such as `[Page 3]`.
- If page markers appear inside a paragraph, move them to a nearby standalone line only when doing so improves readability without changing order.

## Figures and Tables

- Preserve figure/table numbers exactly, such as `Figure 2`, `Table 1`, `Fig. 3`.
- Translate captions and notes.
- Preserve subfigure labels, column labels, units, and statistical notation.
- Do not invent missing visual content. If the extracted source only contains a caption, translate the caption.

## Formulas and Equations

- Preserve formulas unchanged.
- Preserve equation numbers unchanged.
- Preserve variable names, Greek letters, mathematical operators, and units.
- Translate surrounding explanatory text.

## Citations

- Preserve citation markers exactly:
  - Numeric: `[1]`, `[2, 5]`, `[3-7]`
  - Author-year: `(Smith, 2020)`, `Smith et al. (2020)`
  - Legal or standards-style references when present
- Do not renumber citations.

## Footnotes and Endnotes

- Translate footnote text.
- Preserve footnote numbers or markers.

## References or Bibliography

- Do not translate reference entries.
- Translate the section heading as `参考文献` when useful, but do not change any reference entry.
- Copy all reference entries unchanged.
- Preserve DOI, URL, arXiv id, publisher, venue, title, author, and year text.

## Code, Algorithms, and Pseudocode

- Preserve code blocks and pseudocode syntax.
- Translate surrounding explanations, comments only if they are prose comments and translation does not break syntax.
- Preserve algorithm names, line numbers, variable names, and identifiers.
